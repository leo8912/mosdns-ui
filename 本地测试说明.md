# MosDNS 面板本地开发与测试指南

如果你不想立即运行 Docker，可以在本地手动启动前端和后端进行测试。

## 1. 启动后端 (Backend)
后端负责数据采集和 API 提供。

```bash
cd backend
# 首次运行请确保已安装依赖: npm install
# 修改 .env 或直接运行 (默认连接 http://127.0.0.1:9080/metrics)
node index.js
```
*   **API 地址**: `http://localhost:3000`
*   **数据库**: 会在 `../data/mosdns.db` 自动生成。

## 2. 启动前端 (Frontend)
前端提供可视化界面。

```bash
cd frontend
# 首次运行请确保已安装依赖: npm install
npm run dev
```
*   **访问地址**: `http://localhost:5173` (Vite 默认端口)
*   **注意**: 前端代码中默认连接的是 `localhost:3000`。

---

## 3. Docker 错误修复说明
刚才报错是因为 `Dockerfile` 第 16 行多了一个冒号：
*   ❌ `--from:frontend-build`
*   ✅ `--from=frontend-build`

我已经修复了该文件。你现在可以再次尝试运行：
`docker-compose up -d --build`
